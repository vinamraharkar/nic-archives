## Co-operative Banking in Place - COIN

Ever since Co-operative banking came into existence, it has been a landmark in bridging the gap between the banking sector and the common people especially in the rural sector. It has indeed played a pivotal role in socio-economic upliftment of the poor and the downtrodden in our country.

But to compete with the increasing day to day competition in the financial sector, NIC has developed specialised software named COIN (Cooperative Banking in Place). Read ahead to know more…

COIN-An Overview

COIN (Co-operative Banking In- Place) The first ICT based solution by the government for Total Branch Automation (TBA) making effective economic reforms in Co-operative Bank and its branches thereby enhancing the reach of the bank to the Rural Masses for the prosperity of the State. COIN has set a new standard of citizen-centric services by supporting Commercial banking along with agricultural loans, automated credit and disbursement of pension to freedom fighters. The business reengineering model of Group accounting for the state government with MICR interface of cheque clearing with RBI paved the way for COIN to achieve Microsoft Award in 2006. A mark jump of 10,000 customers in single year has shown the trust on the new system and the stake holder bank created a history by touching the height of 114 crore Net Profit against the previous net profit of only one crore. This solution was replicated in all branches of Bihar State Cooperative bank Ltd. (BSCB) across the state.

Important Technical Features of COIN

COIN is developed using COM (Component Object Model), which is general architecture that lays the foundation of the project. It uses technology of building inter operable component through an object orientation. An arbitrary number of OLE Custom Controls (OCXs) are developed and used as fabricated components to aid in building the project. A Dynamic Link Library (DLL) file is developed which groups many functions and class modules that are compiled, linked and stored separately from project facilitating faster response and easier maintenance.

Architecture of COIN

Architectural view of COIN

COIN supports 3-tier architecture -

1. Presentation Layer -Is responsible for the presentation of data, receiving user events and controlling the user interface. COIN Software's executable file resides in presentation layer containing Presentation logic having User Interface, displaying data to the user, accepting input from the user facility.

2. Business Layer-Business-objects that implement the business rules “live” here, and are available to the client- tier. This tier protects the data from direct access by the clients. Custom Dynamic Link Library (DLL) contains business logic of Bank, is middle-tier component. It contains Business logic, which means Data Validation, ensuring the data is correct before being added to the database.

3. Data Layer-It contains Data Access Logic which means Database Communication, accessing tables and indices, packing and unpacking data.

Logical view of the COIN Software

One of the notable features of COIN software is that it provides integration with MICR Encoder Machine, which is being used for MICR Inward & Outward clearing of Cheques.

COIN software also enables user to get passbook updated using Passbook Printer. Software is smart enough to help Branch Manager monitoring the activities of its employees working in Branch along with giving correct and up-to-date position of cash at bank with denomination details of currency. It instantly prepares Cashbook, trial balance and Balance Sheet. It prints several types of vouchers at for Assistants at the time of passing the transaction. Chest and Cash box of bank is also computerised which is very useful in cash management. Cashiers use software to collect and disburse the cash from and to the customers. Management of Grouping for the A/c is available in this software. Verification of the Signature & Photograph of customers is also available. Instead of passing related transaction one by one, this software allows the user to pass all the related transaction at a single keystroke. This facility helps the user to generate correct Cash Book at day end. COIN prepares pictorial analysis of several MIS reports for decision-making. Application and data access & Information Management is neatly designed. Backup & Recovery process is built-in with COIN Software.

Coin software is very secure software system, which provides five levels of Application Security Management viz. Operating System Level, RDBMS Level,Application Level, Transaction Level & Audit Log Level.
